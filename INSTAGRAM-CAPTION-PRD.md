# 인스타 캡션 생성기 PRD

## 제품 개요
뷰티샵 원장님을 위한 인스타그램 캡션 자동 생성 기능
블로그라이터의 두 번째 콘텐츠 생성 도구

### 핵심 가치
- **반영구 무드(진지/감성/노하우)**를 모든 뷰티 업종에 적용
- 다른 샵들이 다 비슷한 감성 톤일 때 차별화
- 9가지 유형 + 업종별 키워드 적용

---

## 1. 학습한 캡션 패턴 (총 20개 분석)

### 9가지 캡션 유형

| # | 유형 | 길이 | 핵심 구조 | 마무리 이모지 |
|---|------|------|----------|--------------|
| 1 | **노하우형** | 중-장 | 통념 → 반전 → 인사이트 | ⭐️ |
| 2 | **고객사례형** | 중 | 사연 → 시술자 판단 → 진심 | 🙏🏻🩵 |
| 3 | **자신감형** | 짧음 | 짧고 도발적, 한 방 | 💃🏼💃🏼 |
| 4 | **철학/신념형** | 매우 김 | 길게 신념 풀기 + 다짐 | 🙏🏻 |
| 5 | **동기부여형** | 중-장 | 명령형 격려, 후배 호명 | 💪🏻x4 |
| 6 | **모객/스토리형** | 김 | 본인 스토리 → 깨달음 → CTA | 😀 |
| 7 | **성장기록형** | 중 | 자기 디스 + 단계 + 다짐 | ㅎㅎ |
| 8 | **일상/커뮤니티형** | 중 | 모임/감사/따뜻 | 😀 |
| 9 | **기술/정보형** | 짧음 | 압축 정보 + 자연 모객 | 없음 |

### 모든 유형의 공통 공식

**필수:**
- ⠀ (빈 점, U+2800) 줄바꿈으로 가독성 ↑
- 한 줄씩 짧게 끊기
- 시술자(원장) 1인칭 시점
- 마지막에 이모지 1~4개

**금지:**
- 가격 직접 명시
- "예약하세요", "DM 주세요" 같은 직접 영업
- 마크다운(##, **) 등

**자연스러운 어필:**
- 마지막 한 줄에 자연 어필 ("수다는 이런 것들부터 교육한다")
- 신념 선언 ("저는 ___하는 사람이 되고 싶습니다")

## 2. 업종별 톤 차이 (참고)

| 업종 | 기존 SNS 톤 | 우리가 적용할 무드 |
|------|-----------|------------------|
| 반영구 | 진지/철학 | 그대로 유지 |
| 네일 | 감성/이모지多 | 반영구 무드 (차별화) |
| 헤어 | 친근/사례 중심 | 반영구 무드 (차별화) |
| 속눈썹 | 자연스러움 강조 | 반영구 무드 (차별화) |
| 피부관리 | 비포-애프터 | 반영구 무드 (차별화) |
| 왁싱 | 위생/유머 | 반영구 무드 (차별화) |
| 두피관리 | 효과 강조 | 반영구 무드 (차별화) |
| 메이크업 | 비주얼 | 반영구 무드 (차별화) |

→ **반영구 무드를 통일된 브랜드 톤으로 사용**해서 차별화 포인트 만들기

---

## 3. 입력값 구조

### 사용자 선택
- **업종**: 반영구/네일/헤어/속눈썹/피부관리/왁싱/두피관리/메이크업
- **캡션 유형**: 9가지 중 선택 (또는 AI 자동 추천)
- **글 길이**: 짧게(3줄) / 보통(10줄) / 길게(20줄+)
- **마무리 이모지**: 자동 / 직접 선택

### 사용자 입력 (선택사항)
- **시술/주제 키워드**: 예 "헤어스트록", "젤네일", "레이어드펌"
- **고객 사연**: 사연 있으면 입력 → 고객사례형
- **본인 스토리**: 본인 스토리 있으면 → 모객/스토리형
- **전달할 메시지**: 핵심 메시지
- **모임/이벤트 정보**: 있으면 → 일상/커뮤니티형

## 4. 출력 형식

```
[유형에 맞는 도입]
⠀
[본문 짧은 문장 다수]
⠀
[인사이트/판단/메시지]
⠀
[감성 마무리 + 이모지]
```

**공통 적용:**
- 한 줄 12~25자 권장
- 빈 줄(⠀)로 단락 구분
- 길이별 자동 조정

## 5. 프롬프트 설계 방향

### 시스템 프롬프트
```
당신은 뷰티샵 원장님이 인스타그램에 올리는 캡션을 작성하는 시스템입니다.

## 글의 정체성
이 글은 시술자(원장님) 1인칭 시점의 인스타 캡션입니다.
다른 샵들의 가벼운 감성 톤이 아니라,
진지함과 전문성, 따뜻함을 담은 신뢰감 있는 톤이어야 합니다.

## 9가지 유형 중 선택된 유형으로 작성
[유형별 구조 주입]

## 줄바꿈 규칙
- 모든 단락 사이에 ⠀ (U+2800) 한 줄 삽입
- 한 줄 12~25자 권장
- 짧게 끊어서 모바일 가독성 ↑

## 금지 표현
- 가격 직접 명시
- "예약하세요", "DM 주세요" 직접 영업
- 마크다운, 해시태그(별도 입력)

## 마무리 규칙
- 감성 멘트 + 이모지 1~4개
- 자연스러운 어필 (직접 영업 금지)
```

### 유형별 사용자 프롬프트 예시

**노하우형:**
```
업종: ___ / 키워드: ___
구조: 통념 → 반전 → 인사이트 → 약속
- 사람들이 흔히 ___라고 생각하는 통념 제시
- 그게 사실이 아닌 이유 반전
- 진짜 중요한 인사이트 제공
- 알게 되면 어떻게 달라지는지 약속
```

**고객사례형:**
```
업종: ___ / 시술: ___ / 사연: ___
구조: 고객 사연 → 시술자 판단 → 결과 → 따뜻한 메시지
```

(나머지 유형도 동일하게 정의)

## 6. UI 흐름

```
1. 콘텐츠 유형 선택
   - 블로그 글 / 인스타 캡션 (신규)

2. 인스타 캡션 선택 시:
   ├ 업종 선택
   ├ 캡션 유형 선택 (9가지) or AI 추천
   ├ 시술 키워드 입력
   ├ 추가 정보 입력 (사연/스토리/메시지 등)
   ├ 길이 선택
   └ 생성 → 복사

3. 결과 화면
   - 생성된 캡션 표시
   - 복사하기 버튼
   - 다른 유형으로 재생성 버튼
   - 해시태그 추천 (기법명 위주)
```

## 7. 가격 정책 (제안)

### 옵션 A: 통합 인상
- 블로그 + 인스타 캡션 → 월 14,900원 → 19,900원

### 옵션 B: 추가 모듈
- 블로그 (기존): 9,900원
- 블로그 + 인스타 (추가): 14,900원

### 옵션 C: 무료 보너스
- 블로그 구독자에게 인스타 캡션 무료 제공
- 신규 가입 유도용

→ **추천: 옵션 C**로 시작 후 사용자 반응 보고 분리

## 8. 개발 우선순위

### Phase 1 (MVP)
- 9가지 유형 중 가장 자주 쓰이는 4가지 (노하우/고객사례/자신감/모객)
- 8개 업종 모두 지원
- 기본 입력 폼 + 생성 + 복사
- 반영구 무드로 통일

### Phase 2 (확장)
- 9가지 유형 전체
- 업종별 키워드 사전 추가
- 해시태그 자동 추천
- 생성 결과 히스토리

### Phase 3 (고도화)
- AI 자동 유형 추천
- 캡션 + 카드뉴스 이미지 동시 생성
- 인스타 직접 발행 (Meta API 연동)
